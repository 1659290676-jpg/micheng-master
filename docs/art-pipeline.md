# 猫咪美术资产：在哪个环节「投喂」

原则：**先灰盒 + 占位图跑通玩法，再按模块替换为你的猫咪资产**，避免一开始就被资源阻塞。

## 投喂时机总表

| 阶段 | 模块 | 何时需要你给图 | 格式建议 |
|------|------|----------------|----------|
| **M1** | 棋盘与格 | 六边形格 3 态（己方/敌方/中立）+ 可选草地底图 | PNG 透明，边长 2 的幂次（如 128×148） |
| **M1** | UI 壳 | 金币、计时器底框（可后补） | PNG 9-slice 或整图 |
| **M2** | 问号格 | 问号盲盒格图标（猫咪主题，如「神秘猫箱」） | PNG |
| **M2** | 建筑 | 箭塔、兵营、矿山各 1 套（可含 1 级） | PNG，锚点底部中心 |
| **M3** | 战斗单位 | 每个兵种：待机 + 行走（2~4 帧）+ 可选攻击 | 精灵表或逐帧 PNG |
| **M3** | 防御塔 | 塔身 + 可选攻击特效 | PNG |
| **M4** | 卡牌 | 卡面竖版（比例约 3:4） | PNG |
| **M4** | 局外 | 宝箱 4 档、主城、木材/金币图标 | PNG |
| **M5** | 领地 | 扩地格、领主猫立绘 | PNG |

## 目录约定（你投喂时直接丢这里）

```
assets/
  cats/                    # 你的原创猫咪 IP（按模块分子文件夹）
    board/                 # M1 格子
    buildings/             # M2 建筑
    units/                 # M3 单位（每个单位一个子文件夹）
    cards/                 # M4 卡面，文件名 = cardId
    ui/                    # 通用 UI、货币、宝箱
    meta/                  # M5 主城、领地
  placeholders/            # 开发用占位（可先由程序生成色块）
```

**命名**：`{cardId}.png` 或 `{unitId}_walk_01.png`，与 `data/cards.json` 的 `id` 一致。

## 不要一开始就全给

1. **M1 只要 3 张格图** 即可开工棋盘。  
2. 单位可以 **1 个兵种** 先跑通自动移动与占格。  
3. 卡面可以最后换，MVP 用稀有度色框 + 文字。

## 技术接入方式（网页版）

- 引擎：**Phaser 3**（`src/web`）通过 `preload` 加载 `assets/cats/...`。  
- 配置里只写 `id`，程序拼路径：`assets/cats/units/${id}/atlas.json` 或单图。  
- 你更新 PNG 后 **无需改代码**，刷新页面即可（开发模式）。
