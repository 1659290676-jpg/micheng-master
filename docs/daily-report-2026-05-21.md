# 《咪城大师》工作日报

日期：2026-05-21  
项目：咪城大师  
当前重点：从 Phaser 网页原型迁移到 Laya3D 版本

## 一、昨日工作进度

### 1. Laya3D 迁移推进

昨日继续推进《咪城大师》从 Phaser 网页原型向 Laya3D 的迁移工作。当前采用“新增独立咪城模块 + 逐步替换旧玩法”的方式，不复用原 Laya 项目中的恐龙/城市玩法逻辑。

已完成并验证的阶段包括：

- 第一阶段：Laya3D 空棋盘接入，第一关 9×16 平顶六边形棋盘可在 LayaAir 预览中显示。
- 第二阶段：基础实体占位接入，主塔、基础建筑和士兵占位表现已能出现。
- 第三阶段：图纸建造首版接入，主塔/建筑周边可出现贴地图纸，点击后建筑从底部翻转生成，并刷新周边图纸。
- 第四阶段：HUD 与经济首版接入，已显示金币、倒计时、占格数和提示信息，建造会扣金币，金矿会提高金币增长。
- 第五阶段：战斗与占地首版接入，士兵可从兵营附近出发，按格移动、占地，并攻击敌方士兵或建筑；主塔和防御塔可攻击士兵。

### 2. 视觉与交互调整

昨日重点修正了 LayaAir 预览中的若干表现问题：

- 调整了相机为竖屏 720×1280 取景，更接近手机游戏画面。
- 调整了绿方在近端、红方在远端的棋盘方向。
- 移除了上一阶段用于测试图纸点击层级的试验小人，避免干扰正式士兵系统观察。
- 修正图纸金币数字显示问题，目标是让 `25 / 50 / 100 / 250` 在当前运行视角下更容易识别。
- 修正士兵攻击敌方单位或建筑时占地不明显的问题：攻击相邻敌方目标时，会同步把自己脚下和目标所在格染成己方颜色。

### 3. 版本与仓库整理

昨日已将当前 Laya3D 迁移阶段成果单独保存到公开 GitHub 仓库，便于后续与原 Phaser 网页版本区分管理。

- Phaser 网页原型仓库：<https://github.com/1659290676-jpg/micheng-master>
- Laya3D 迁移仓库：<https://github.com/1659290676-jpg/ccfirst-laya3d>

## 二、今日计划工作内容

### 1. 继续验收第五阶段：战斗与占地

今日优先继续检查 Laya3D 版本中的士兵战斗闭环：

- 验证士兵是否稳定从兵营相邻可走格出发。
- 验证士兵是否按格移动，不穿过实体建筑。
- 验证士兵走到或攻击敌方地块时，地块颜色是否正确变为己方。
- 验证士兵攻击敌方士兵、敌方建筑时是否符合预期。
- 验证防御塔和主塔是否只造成伤害，不阻止士兵继续移动。

### 2. 补齐胜负判定

在战斗与占地稳定后，计划接入胜负判定：

- 主塔被毁后触发失败/胜利。
- 3 分钟倒计时结束后，根据双方占格数量判断胜负。
- 初步接入结果提示，为后续结算界面做准备。

### 3. 准备第六阶段：敌方 AI

当前红方暂时不自动扩建。今日如果第五阶段验收顺利，将开始设计或接入红方 AI 的最小版本：

- 红方可基于已有建筑周边图纸自动建造。
- 红方可产生士兵并向玩家方向推进。
- 保持 AI 行为简单可控，先用于验证完整对战闭环。

## 三、当前风险与注意事项

- Laya3D 版本仍处于迁移验证阶段，当前目标是先做出离线可玩的完整 3D 闭环。
- 当前 `Main.ts` 仍是临时调试启动模式，用于直接进入咪城棋盘，后续需要替换成正式入口。
- Laya3D 工程中仍包含旧框架内容，当前原则是不直接大改旧玩法模块，优先保持咪城模块隔离。
- 部分战斗数值为阶段性调试值，后续需要统一迁移到配置表。
