# 关卡数值模块（统筹）

> 所有可调数值集中在 `data/`，改平衡只改 JSON，不改玩法代码。

## 文件职责

| 文件 | 内容 |
|------|------|
| `data/match_config.json` | 局时长、开局金、地图 9×16、绿红分界行 |
| `data/mystery_pool.json` | 25 金问号池权重、保底、30 秒无传说 |
| `data/barracks_pools.json` | 50 / 100 / 250 金兵营出货权重 |
| `data/buildings.json` | 固定建造：金矿、剑塔（100% 确定） |
| `data/cards.json` | 每张卡的建造/生产/战斗数值 |
| `data/deck_starter.json` | 初始 8 张卡组 ID 列表 |

## 单卡数值字段（士兵/兵营类）

| 字段 | 含义 |
|------|------|
| `buildingHp` | 兵营建筑血量 |
| `produceIntervalSec` | 生产一只单位的间隔（秒） |
| `unitHp` | 产出单位血量 |
| `attack` | 攻击力 |
| `moveSpeed` | 移动速度（格/秒，实现时换算） |
| `attackRange` | 攻击范围（格） |
| `targets` | `ground` / `air` |

## 固定建筑字段

| 建筑 | 花费 | 建筑血量 | 其他 |
|------|------|----------|------|
| 金矿 Lv1 | 50 | 140 | 产金间隔 5.995s，+10 金 |
| 剑塔 Lv1 | 50 | 400 | 攻击 120，间隔 2.995s |

## 随机池（已按你确认写入）

- **问号 25 金**：空 70% / 普通 10% / 史诗 15% / 传奇 5%
- **兵营 50 金**：普通 74% / 稀有 20% / 史诗 5% / 传奇 1%
- **兵营 100 金**：普通 50% / 稀有 40% / 史诗 8% / 传奇 2%
- **兵营 250 金**：史诗 90% / 传奇 10%

## 平衡迭代流程

1. 在 `data/cards.json` 改数值 → 刷新网页测试  
2. 对战感觉「推不动/秒破」→ 只调 `unitHp` / `attack` / `produceIntervalSec`  
3. 经济太快/太慢 → 调 `mine` 与 `match_config.startingGold`
